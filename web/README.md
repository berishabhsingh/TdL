# Web Interface

This directory contains the Flask-based web application component of the LeechxTool bot.

## Overview

The web application serves multiple purposes, but its primary function is providing an interface for users to select specific files from torrents being downloaded via Aria2 or qBittorrent before the download starts or completes.

## Components

- **`wserver.py`**: The core Flask application script. It sets up routes, renders HTML templates, and handles POST requests from the web interface to update file selection priorities in Aria2 or qBittorrent.
- **`nodes.py`**: Contains helper functions, particularly `make_tree`, which recursively parses file lists from the torrent clients and generates HTML for the tree-view file selector in the web UI.

## File Selection (Torrent)

When a user initiates a torrent download (often requiring specific file selection), they are provided with a link and a pin code generated by the bot. This link points to the `/app/files/<id>` route managed by `wserver.py`.

1.  **Authentication**: Users must enter the correct pin code to access the file selection page.
2.  **Tree View**: The page presents a hierarchical tree view of the torrent's contents, generated dynamically.
3.  **Selection**: Users can select or deselect individual files or entire folders.
4.  **Submission**: Upon submission, the web app updates the respective torrent client (Aria2 or qBittorrent) to pause unselected files and prioritize selected ones.
